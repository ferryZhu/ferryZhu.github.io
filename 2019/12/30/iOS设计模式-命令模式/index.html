iOS设计模式-命令模式

发表于 2019-12-30 | 分类于 设计模式 |

一、简介

  1. 定义

    命令模式将“请求”封装成对象，以便使用不同的请求、队列或者日志来参数化其他对象。实现请求和实现的二者之间的松耦合，命令模式也支持可撤销的操作。

  2. 核心对象
    • 调用者(Invoker): 持有一个命名对象，并在某个时间点调用命令对象的execute方法。
    • 命令对象(commend): 持有一个接受者，和一个execute方法，在execute值中执行接收者的动作。
    • 接收者(Receiver): 动作的执行者

二、适用场景

  • 系统需要将请求调用者和请求接收者解耦，使两者不直接交互。
  • 系统需要在不同的时间指定请求、将请求排队和执行请求。
  • 系统需要支持命令的撤销(Undo)操作和恢复(Redo)操作。
  • 系统需要将一组操作组合在一起，即支持宏命令。

三、具体实现

现在有一个遥控器，每一对开关按钮都可以设置不同的功能，比如打开关闭卧室的灯，开启关闭车库的门。需要将遥控器与灯、门等分离，遥控器不需要知道具体的接收者，只需要按下按钮发送命令。

  1. Commend类，用来声明执行操作的接口，含有执行和撤销方法

  2. 接收者，具体的动作执行者（如：灯、门等）

  3. ConcreteCommend，具体的命令类，持有接收者。(灯有两个具体的命令类：开灯命令、关灯命令)

  4. 调用者Invoker类，将命令分发给不同的具体命令类

  5. Client，设置遥控器对应的命令

四、优缺点

  • 优点

    • 降低了对象之间的耦合度
    • 高扩展性，新的命令可以很容易的加入到系统中。
    • 可以调用同一个方法实现不同的功能。

  • 缺点

    • 会产生大量的具体命令类。
